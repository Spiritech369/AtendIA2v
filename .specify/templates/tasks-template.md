# Tasks: [FEATURE NAME]

**Input**: `spec.md`, `plan.md`, `Arquitectura-Deseada.md`, and Product-First constitution.

## Format

- `[ ] T### [P?] [Phase] Task description`
- `[P]` means parallel-safe.
- Every implementation phase must include tests, changed-behavior coverage, verification, and Codex code review.

## Required Task Gates

- [ ] Confirm Definition of Ready before implementation starts.
- [ ] Write unit/integration tests for new or modified behavior.
- [ ] Run relevant tests and report results.
- [ ] Report 100% coverage for new or modified behavior.
- [ ] Run Codex code review against base branch or uncommitted changes.
- [ ] Update Feature Readiness and Definition of Done evidence.

## Tasks

- [ ] T001 [Phase] Replace this template with approved implementation tasks.
